Module evolve 

Source
Expand description

Evolve: dynamic markdown file for conversation history and action log (Phase 3). Path from DAL_AGENT_CONTEXT_PATH or [agent] context_path in agent.toml / dal.toml; default ./evolve.md.

Functions§

append_conversation
Append a conversation turn (user message + agent response) to the context file.
append_log
Append an action log entry (e.g. sh::run, command, result).
append_log_timed
Append an action log entry with task timing fields.
append_summary
Append a summary section (e.g. periodic or on-demand session summary).
append_working_memory_line
Append one bullet under ## Working memory (created before ## Conversation if missing). Use for decisions, paths, open threads — short lines (sanitized, capped) that actually evolve context.
get_path
Return the configured context file path (for display or inclusion).
load
Load the current context file content. Creates file with header if missing.
load_recent
Load recent context: last max_lines lines of the file, or full content if max_lines <= 0. Use for inclusion in agent context without loading the entire history.
load_recent_for_prompt
Prompt-oriented evolve load: working memory bullets, last N conversation turns, optional summary tail — excludes the action-log table by default (noisy for reasoning).
sanitize_for_conversation
Sanitize content for safe inclusion in the conversation log (prevents log injection).
trim_retention
Trim the context file to keep only the last keep_tail_lines lines of content (after the header). Preserves the “# Agent context” and “## Conversation” header, then keeps the last keep_tail_lines lines.
