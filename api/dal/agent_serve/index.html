Agent local server: dal agent serve [name] [--port N] [--mold path] One agent per process; HTTP API for status, message, messages, task, tasks, health. See docs/development/AGENT_LOCAL_SERVER_DESIGN.md.

Structs§

AgentServeState 🔒
Shared state for the agent server: the single agent’s context. When prompt_only is true (no behavior script), the server responds to each message via the AI and posts the reply.
MessageBody 🔒
POST /message — body: { sender_id, content } or { sender_id?, message_type?, content }; optional objective, sub_tasks (P2), include_dal_summary (P3), working_root (Phase D).
TaskBody 🔒
POST /task — body: { task_id?, description, priority?, requester_id? }; optional sub_tasks (P2), include_dal_summary (P3). In prompt-only mode, requester_id (optional) is used to send the task result back as a message.
